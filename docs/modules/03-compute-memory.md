# 3. 计算、内存与显存构成

> 状态：占位草稿。

## 在 MLSys 地图中的位置

- 生命周期位置：训练和推理都会遇到的底层资源问题。
- 系统层次：算子与内存、框架运行时、单卡显存预算。
- 主要资源：计算、显存、内存带宽、临时 buffer。
- 典型症状：OOM、显存峰值异常、模型参数不大但实际占用很高、profiler 里出现 memory-bound 算子。
- 关联模块：模块 2 的 profiler，模块 7 的训练优化，模块 8 的并行策略，模块 10 的 KV cache。

## 核心问题

profiler 里看到的算子瓶颈从哪里来？显存不够装时，系统到底装了什么？

## 这篇文章要补什么

- 从成熟 workload 的 profiler 结果里挑一个 tensor op 或 matmul 现象。
- 解释计算量、内存访问、带宽、算子形态和实际耗时之间的关系。
- 避免一上来手写 kernel，把矩阵乘作为理解瓶颈的显微镜。
- 拆清楚参数、梯度、优化器状态、activation、临时 buffer、KV cache 分别怎么吃显存。
- 讨论显存峰值、碎片、OOM 诊断，以及为什么“模型文件大小”不等于运行时显存占用。
