# Hello MLSys

这是一个很早期的 MLSys 课程仓库，目前处在 Alpha 草稿阶段。

现在这里更像一张学习地图、一份课程骨架和一批待补的章节占位。它还不是完整教材，也不是正式 syllabus。很多章节还只有标题，实验、案例和参考资料会慢慢补。

## 这是什么

这门课想帮大家用工程视角理解现代 AI 工作负载背后的系统问题。

核心主线是：

```text
成熟系统 -> 观察现象 -> 测量指标 -> 反推瓶颈 -> 剥开机制 -> 生产决策
```

也就是说，我们不急着背工具名，不从零造一个玩具系统，也不把课程写成工具清单。每个模块都尽量从成熟系统里的现象开始，比如训练 step 为什么慢、服务 P95 为什么变差、显存为什么不够、batch 为什么改变吞吐和延迟。然后再用指标和实验去解释它。

## 适合谁

适合这些同学：

- 学过一点深度学习，想知道模型跑起来以后系统在忙什么
- 做过训练或推理 demo，想补性能分析和部署直觉
- 对 AI Infra、训练平台、推理服务、MLOps 感兴趣
- 想从“会调包”往“能解释为什么慢，知道怎么取舍”走一步

默认你会写基本 Python，知道 PyTorch 或类似框架的基本用法，听过训练、推理、batch、显存这些词。不要求有大 GPU 集群，也不会把低成本 demo 包装成真实大规模生产经验。

## 本地运行文档

需要先安装 pnpm 和 Node.js。

```bash
pnpm install
pnpm docs:dev
```

构建静态文档：

```bash
pnpm docs:build
```

本地预览构建结果：

```bash
pnpm docs:preview
```

## 怎么贡献

欢迎贡献章节草稿、实验设计、实验结果、图表、错别字修正和参考资料整理。

贡献时请尽量沿着这条格式写：

```text
成熟系统 -> 观察现象 -> 测量指标 -> 反推瓶颈 -> 剥开机制 -> 生产决策
```

不要把章节写成工具列表，也不要写成八股问答。比如不要只介绍 vLLM 有哪些参数，而是解释某个现象：为什么并发上来以后 P95 变差，哪些指标能看出来，KV cache、batching、排队和显存分别在影响什么，最后如果是一个线上服务该怎么取舍。

更多约定见：

- [贡献指南](docs/guide/contributing.md)
- [仓库结构与内容约定](docs/guide/repo-structure.md)
- [章节模板](docs/guide/chapter-template.md)

## 还没承诺什么

这个项目还很早，不承诺每个模块都已经可学习，也不承诺覆盖所有工具、论文和生产场景。课程会尽量提供低算力路径，GPU 内容会作为选做或进阶路径出现，不把资源条件变成入门门槛。
